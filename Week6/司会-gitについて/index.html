１.Gitとは

概要：ソースコードを管理するためのツール。

ソースコード：html/css/js

ターミナルの上で操作する

できること

変更したデータ・履歴を保存管理することができる
誰がいつどこを編集したのかがわかる

実際に使ってみる

$ git init

意味：リポジトリを新規に作成するときに使用するコマンド

リポジトリ：システムの情報を管理する貯蔵庫。

$ git add「ファイル名」

意味：保存するファイルを選択するコマンド

「.」は「現在のディレクトリ」を表す

$ git commit -m "コミットメッセージ"

意味：git addされた範囲をコミット(保存)する

コミットメッセージでは共同開発の際にコミットした内容を書くと他メンバーに伝わりやすい

Git Graph

概要：vscode上でgitの履歴を見ることができる拡張機能

git add と git commitで更新される

ブランチ

概要：セーブデータを複数個作ることができるもの

書き方

$ git branch "ブランチ名"

※ブランチを作っただけで移動していない

$ git checkout "ブランチ名"

ブランチに移動することができる

ブランチの内容を1つにする

別々のブランチで書いた内容を1つにしたい

書き方

$ git checkout "ブランチ名"

概要：別ブランチに移動するコマンド

$ git merge "取り込みたいブランチ名"

概要：「取り込みたいブランチ名」の内容をマージするコマンド。
