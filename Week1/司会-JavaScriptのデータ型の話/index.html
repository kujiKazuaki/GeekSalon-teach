１.値とは

意味：何らかの式を評価した結果

例

let count = 0
console.log("こんにちは")
if (30 % 3 === 0) {}
const func = function() {}

イメージ：変数に代入するときの右側。if文のかっこの中

２.データ型と種類

  • 数値
  • 文字列
  • 論理型(真偽値)
  • 関数
  • undefined
  • 配列
  • オブジェクト

数値

意味：「数字」に当たるもの

例： 1, 0.1,

書き方：半角数字で書く

文字列

意味：「""」や「''」で囲ったもの

例：console.log("こんにちは") => こんにちは

書き方１：「""」や「''」で囲う

※「""」：ダブルクォーテーション

※「''」: シングルクォーテーション

※「""」で囲ったものは数字であっても文字列に変換される！

書き方２：「``」で囲い、変数を使う場合は${ 変数 }とかく

※「``」：バッククォーテーション

論理型(真偽値)

意味：「true」「false」で審査されるもの

イメージ：「もし〇〇だったら」

例：if(真偽値)

書き方：値に「true」か「false」と書く

trueとfalseは比較演算子=== で間接的に使う

関数

意味：文の集まりを１つの処理としてまとめる機能

　　　関数も値のひとつ！

書き方１：function() { 関数が呼び出された時の処理 }

書き方２：function(引数) { 関数が呼び出された時の処理 }

※変数定義した場合、「変数()」という書き方をする必要がある

メリット：なんども使える

※引数を設定した場合、「変数(引数)」という書き方ができる

引数のメリット：引数を変えるだけで同じ処理をしてくれるのでコード書くの楽

書き方３：「return(返り値)」を使う

返り値：「戻り値」ともいう。関数内で処理した結果を「返り値」として返すことができる

メリット：返り値を使って再利用できる => if文の条件式, 別の関数の引数に入れ込む

undefined

意味：「定義されてない」という意味。

例：let 〇〇,=> 変数を宣言したが、一度も使われてない時

配列

意味：値に番号をつけて１か所に格納できる型

書き方：[要素, 要素,]

要素 => 配列・文字列・オブジェクトなどの値が入る

メリット：複数の文字列や数値などの値をまとめたいとき

※呼び出すときは「変数名[インデックス番号]」という書き方をする

インデックス番号は絶対「0」から始まる!

オブジェクト

意味：値にラベルをつけて格納できる型

書き方：{ key: value, key: value, ...}

※keyに当たる部分は「文字列」か「数字」

※valueに当たる部分はどんな値を入れてもいい

※呼び出すときは「変数名.key」という書き方をする

メリット：値を名前( キー ) と紐付けてまとめておける

３.配列とオブジェクトの違い

配列 オブジェクト
書き方 [要素, 要素,] { key: value,}
呼び出し方 変数名[インデックス番号] 変数名.key
特徴(区別化) 順番を意識しないといけない 順番を意識しなくていい
